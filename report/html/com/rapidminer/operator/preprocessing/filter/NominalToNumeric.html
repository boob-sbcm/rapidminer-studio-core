58    /**
59    * This operator maps all non numeric attributes to real valued attributes. Nothing is done for
60    * numeric attributes, binary attributes are mapped to 0 and 1.
61    *
62    * For nominal attributes one of the following calculations will be done:
63    * <ul>
64    * <li>Dichotomization, i.e. dummy coding or effect coding: one new attribute for each but one value
65    * of the nominal attribute. The new attribute which corresponds to the actual nominal value gets
66    * value 1 and all other attributes gets value 0.</li> If the nominal value is the one for which no
67    * attribute is being created, all other target attributes are set to 0 (dummy coding) or -1 (effect
68    * coding).
69    * <li>Alternatively the values of nominal attributes can be seen as equally ranked, therefore the
70    * nominal attribute will simply be turned into a real valued attribute, the old values results in
71    * equidistant real values.</li>
72    * </ul>
73    *
74    * At this moment the same applies for ordinal attributes, in a future release more appropriate
75    * values based on the ranking between the ordinal values may be included.
76    *
77    * @author Ingo Mierswa, Sebastian Land, Marius Helf
78    */
