33    /**
34    * DirectedGGA is an acronym for a Generating Genetic Algorithm which uses probability directed
35    * search heuristics to select attributes for generation or removing. Its approach to generating new
36    * attributes differs from the original one and is the same as the one of {@link YAGGA}. <br/>
37    *
38    * The (generating) mutation can do one of the following things with different probabilities:
39    * <ul>
40    * <li>Probability {@rapidminer.math p/4}: Add a newly generated attribute to the feature vector
41    * </li>
42    * <li>Probability {@rapidminer.math p/4}: Add a randomly chosen original attribute to the feature
43    * vector</li>
44    * <li>Probability {@rapidminer.math p/2}: Remove a randomly chosen attribute from the feature
45    * vector</li>
46    * </ul>
47    * Thus it is guaranteed that the length of the feature vector can both grow and shrink. On average
48    * it will keep its original length, unless longer or shorter individuals prove to have a better
49    * fitness.<br/>
50    *
51    * In addition to these mutation heuristics probablilities based on the weights of the attributes
52    * are calculated. It is more likely for attributes with a great weight to be selected for
53    * generating new attributes. On the other hand the probability for removing an attribute from the
54    * example set will decrease for attributes with great weights. This decreases the amount of needed
55    * generations drastically. <br/>
56    *
57    * Another enhancement in comparison to the original GGA is the addition of several generators like
58    * the ones for trigonometric or exponential functions. In this way a sinple linear working learning
59    * scheme which can deliver weights can be used as inner operator. If this learner can also estimate
60    * its performance it is not longer necessary to use a inner cross-validation which also decreases
61    * learning time. Such a learner is for example the
62    * {@link com.rapidminer.operator.learner.functions.kernel.JMySVMLearner} which delivers the
63    * xi-alpha performance estimation at least for classification tasks. <br/>
64    * .
65    *
66    * Summarized the advantages of this feature construction algorithm are smaller runtimes and smaller
67    * attribute sets as result. These attribute sets increase performance and can be used to explain
68    * the models of more complex learning schemes like SVMs. The additional generators allow the
69    * construction of features which are not possible by the known kernel functions. <br/>
70    *
71    * Since this operator does not contain algorithms to extract features from value series, it is
72    * restricted to example sets with only single attributes. For (automatic) feature extraction from
73    * values series the value series plugin for RapidMiner written by Ingo Mierswa should be used. It
74    * is available at <code>http://rapidminer.com</code>.
75    *
76    * @author Ingo Mierswa
77    */
