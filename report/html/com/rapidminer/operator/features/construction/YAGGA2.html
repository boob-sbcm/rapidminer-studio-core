44    /**
45    * <p>
46    * YAGGA is an acronym for Yet Another Generating Genetic Algorithm. Its approach to generating new
47    * attributes differs from the original one. The (generating) mutation can do one of the following
48    * things with different probabilities:
49    * </p>
50    * <ul>
51    * <li>Probability {@rapidminer.math p/4}: Add a newly generated attribute to the feature vector</li>
52    * <li>Probability {@rapidminer.math p/4}: Add a randomly chosen original attribute to the feature
53    * vector</li>
54    * <li>Probability {@rapidminer.math p/2}: Remove a randomly chosen attribute from the feature
55    * vector</li>
56    * </ul>
57    * <p>
58    * Thus it is guaranteed that the length of the feature vector can both grow and shrink. On average
59    * it will keep its original length, unless longer or shorter individuals prove to have a better
60    * fitness.
61    * </p>
62    *
63    * <p>
64    * In addition to the usual YAGGA operator, this operator allows more feature generators and
65    * provides several techniques for intron prevention. This leads to smaller example sets containing
66    * less redundant features.
67    * </p>
68    *
69    * <p>
70    * Since this operator does not contain algorithms to extract features from value series, it is
71    * restricted to example sets with only single attributes. For (automatic) feature extraction from
72    * values series the value series plugin for RapidMiner should be used.
73    * </p>
74    *
75    * <p>
76    * For more information please refer to
77    * </p>
78    * <p>
79    * Mierswa, Ingo (2007): <em>RobustGP: Intron-Free Multi-Objective Feature Construction</em> (to
80    * appear)
81    * </p>
82    *
83    * @author Ingo Mierswa
84    */
