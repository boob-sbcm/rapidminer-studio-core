35    * This operator finds the optimal values for a set of parameters using a grid search. The parameter
36    * <var>parameters</var> is a list of key value pairs where the keys are of the form
37    * <code>operator_name.parameter_name</code> and the value is either a comma separated list of
38    * values (e.g. 10,15,20,25) or an interval definition in the format [start;end;stepsize] (e.g.
39    * [10;25;5]). Alternatively a value grid pattern may be used by [e.g. [start;end;no_steps;scale],
40    * where scale identifies the type of the pattern.
41    * </p>
42    *
43    * <p>
44    * The operator returns an optimal {@link ParameterSet} which can as well be written to a file with
45    * a {@link com.rapidminer.extension.legacy.operator.io.ParameterSetWriter}. This parameter set can be read in
46    * another process using a {@link com.rapidminer.extension.legacy.operator.io.ParameterSetLoader}.
47    * </p>
48    *
49    * <p>
50    * The file format of the parameter set file is straightforward and can easily be generated by
51    * external applications. Each line is of the form <center>
52    * <code>operator_name.parameter_name = value</code></center>
53    * </p>
54    *
55    * <p>
56    * Please refer to section {@rapidminer.ref sec:parameter_optimization|Advanced Processes/Parameter
57    * and performance analysis} for an example application. Another parameter optimization schems like
58    * the {@link EvolutionaryParameterOptimizationOperator} might also be useful if the best ranges and
59    * dependencies are not known at all. Another operator which works similar to this parameter
60    * optimization operator is the operator {@link ParameterIteration}. In contrast to the optimization
