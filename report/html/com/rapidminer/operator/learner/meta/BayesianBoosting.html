52    /**
53    * <p>
54    * This operator trains an ensemble of classifiers for boolean target attributes. In each iteration
55    * the training set is reweighted, so that previously discovered patterns and other kinds of prior
56    * knowledge are &quot;sampled out&quot; {@rapidminer.cite Scholz/2005b}. An inner classifier,
57    * typically a rule or decision tree induction algorithm, is sequentially applied several times, and
58    * the models are combined to a single global model. The number of models to be trained maximally
59    * are specified by the parameter <code>iterations</code>.
60    * </p>
61    *
62    * <p>
63    * If the parameter <code>rescale_label_priors</code> is set, then the example set is reweighted, so
64    * that all classes are equally probable (or frequent). For two-class problems this turns the
65    * problem of fitting models to maximize weighted relative accuracy into the more common task of
66    * classifier induction {@rapidminer.cite Scholz/2005a}. Applying a rule induction algorithm as an
67    * inner learner allows to do subgroup discovery. This option is also recommended for data sets with
68    * class skew, if a &quot;very weak learner&quot; like a decision stump is used. If
69    * <code>rescale_label_priors</code> is not set, then the operator performs boosting based on
70    * probability estimates.
71    * </p>
72    *
73    * <p>
74    * The estimates used by this operator may either be computed using the same set as for training, or
75    * in each iteration the training set may be split randomly, so that a model is fitted based on the
76    * first subset, and the probabilities are estimated based on the second. The first solution may be
77    * advantageous in situations where data is rare. Set the parameter
78    * <code>ratio_internal_bootstrap</code> to 1 to use the same set for training as for estimation.
79    * Set this parameter to a value of lower than 1 to use the specified subset of data for training,
80    * and the remaining examples for probability estimation.
81    * </p>
82    *
83    * <p>
84    * If the parameter <code>allow_marginal_skews</code> is <em>not</em> set, then the support of each
85    * subset defined in terms of common base model predictions does not change from one iteration to
86    * the next. Analogously the class priors do not change. This is the procedure originally described
87    * in {@rapidminer.cite Scholz/2005b} in the context of subgroup discovery.
88    * </p>
89    *
90    * <p>
91    * Setting the <code>allow_marginal_skews</code> option to <code>true</code> leads to a procedure
92    * that changes the marginal weights/probabilities of subsets, if this is beneficial in a boosting
93    * context, and stratifies the two classes to be equally likely. As for AdaBoost, the total weight
94    * upper-bounds the training error in this case. This bound is reduced more quickly by the
95    * BayesianBoosting operator, however.
96    * </p>
97    *
98    * <p>
99    * In sum, to reproduce the sequential sampling, or knowledge-based sampling, from
100    * {@rapidminer.cite Scholz/2005b} for subgroup discovery, two of the default parameter settings of
101    * this operator have to be changed: <code>rescale_label_priors</code> must be set to
102    * <code>true</code>, and <code>allow_marginal_skews</code> must be set to <code>false</code>. In
103    * addition, a boolean (binomial) label has to be used.
104    * </p>
105    *
106    * <p>
107    * The operator requires an example set as its input. To sample out prior knowledge of a different
108    * form it is possible to provide another model as an optional additional input. The predictions of
109    * this model are used to weight produce an initial weighting of the training set. The ouput of the
110    * operator is a classification model applicable for estimating conditional class probabilities or
111    * for plain crisp classification. It contains up to the specified number of inner base models. In
112    * the case of an optional initial model, this model will also be stored in the output model, in
113    * order to produce the same initial weighting during model application.
114    * </p>
115    *
116    * @author Martin Scholz
117    */
