53    /**
54    * When started for the first time, listens on a given socket on localhost. If started for the
55    * second time, contacts this socket and passes command line options to this socket. We use a lock
56    * file mechanism to make sure only one server instance exists.
57    *
58    * The port number on which we listen is stored in a file in the users home directory,
59    * .RapidMiner/rapidminer.lock. The lock file contains two lines: the port number of the first
60    * instance and a random session id. It is readable only to the current user.
61    *
62    * In order to use this class, first try to contact another instance by calling
63    * {@link #sendArgsToOtherInstanceIfUp(String...)}. If true is returned, commands were sent to and
64    * processed by the other instance successfully and we can terminate. If false is returned, the
65    * other instance is not running. In that case, call {@link #installListener(RemoteControlHandler)}.
66    * Now, when another instance is started, callbacks are made to the {@link RemoteControlHandler}
67    * passed. Precisely this is done when calling
68    * {@link #defaultLaunchWithArguments(String[], RemoteControlHandler)}.
69    *
70    * Currently, the only supported message accepted by the server has this format:
71    *
72    * <args session-id="12345678"><arg>arg1</arg><arg>arg2</arg>...</args>
73    *
74    * When opening the socket, the server will respond with <hello/> and confirm success with <ok/>.
75    * Error messages come as <error>message</error>.
76    *
77    * This class is deliberately not using any fancy stuff like RMI to make debugging easy. Just
78    * connect to socket and type to debug. Also, no sophisticated protocol is necessary since this
79    * class will only talk to itself, and never even to a server of another version.
80    *
81    * @author Simon Fischer

83    */
